Hardware Wallet & Digital Signature Flow

How your private key signs a transaction without ever leaving the wallet

1

User Initiates Transaction

User enters receiver card, amount, transaction PIN, and wallet PIN in the frontend.

receiver_card: "4111...0002"
amount: 1000
txn_pin: "0726"
wallet_pin: "0726"
arrow_downward

Output: API payload formed for sender_id=6, receiver_card="4111...0002", amount=1000.

2

Backend Validates PIN

Backend uses bcrypt to verify transaction PIN. If invalid, transaction is rejected.

check_user_pin(user_id, pin)
→ bcrypt.verify(pin, stored_hash)
arrow_downward

Output: txn_pin "0726" verified successfully; request allowed for wallet signing.

3

Wallet Decrypts Private Key

wallet_pin is SHA-256 hashed → used as XOR key → decrypts stored encrypted private key.

key = SHA256(wallet_pin)
private_key = XOR(encrypted_key, key)
arrow_downward

Output: encrypted_private_key=c88b70...02af, key=SHA256("0726"), private_key decrypted in wallet memory.

4

EdDSA Signs Transaction

Message is formed and signed with Ed25519 (EdDSA). Only the signature — not the key — leaves the wallet.

message = "1->2:1000:0"
signature = EdDSA.sign(private_key, message)
arrow_downward

Output: message "6->4111...0002:1000:0" signed; signature generated and exported.

5

Backend Verifies & Commits

Backend fetches user's public key from DB. Verifies signature. If valid → inserts transaction → updates blockchain.

public_key = DB.users[sender_id]
verify(public_key, message, signature) ✓
→ insert TX → update blockchain

Output: verify(6f40ee...75d4, message, signature) = true; transaction committed to blockchain.
